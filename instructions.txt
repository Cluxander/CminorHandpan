✏️ 2. Make changes

Edit your files (for example):
src/HandpanAtlas.jsx
👉 Save → browser updates instantly

🧪 3. Test production version (important)

Before pushing, always do:
npm run build
npm run preview
Open:
http://localhost:4173
👉 This is what Cloudflare will actually deploy

📦 4. Commit your changes

In terminal:
git add .
git commit -m "describe what you changed"
Example:
git commit -m "improve chord layout"

🚀 5. Push to GitHub

git push
👉 This triggers:
automatic build on Cloudflare
automatic deployment

🌐 6. Wait for deployment

Go to:
Cloudflare → Pages → your project → Deployments
You’ll see:
“Building…”
then “Success”

🔄 7. Check your site

Open:
https://your-project.pages.dev
👉 Hard refresh if needed:
Cmd + Shift + R (Mac)
🧠 Visual flow
edit → save → test locally → build → git commit → git push → auto deploy → live
⚠️ Common mistakes (avoid these)
❌ Skipping build test
→ works locally but breaks on deploy
❌ Forgetting git push
→ changes never go online
❌ Cached browser
→ you think it didn’t update
🚀 Optional (but powerful)
🌿 Use branches for safe testing

git checkout -b new-feature
git push origin new-feature

👉 Cloudflare creates:
new-feature.your-project.pages.dev
Test safely without breaking main site.
